# AI 음성 물건등록 구현 메모

## 배포 구성

정적 화면은 기존과 동일하게 동작한다. `/api/transcribe`와 `/api/structure-items`를 지원하는 Node.js 서버리스 환경에 함께 배포하고 `GEMINI_API_KEY`를 서버 환경변수로 설정한다. API가 없거나 실패해도 기존 직접 등록, 사진대지, Excel, ZIP 기능에는 영향이 없다.

브라우저 녹음은 Gemini 공식 지원 형식인 PCM WAV로 생성한다. 최대 60초 녹음은 서버에 영구 저장하지 않고 Gemini API에 인라인 데이터로 전달한다.

## 개인정보 분리

1. 녹음 UI에서 음성 입력 범위를 물건 속성으로 제한한다.
2. 소유자 정보는 기존 기본조사서 폼에서만 입력하도록 안내한다.
3. STT 원문을 구조화 API로 보내기 전 `privacy-filter.js`가 주민등록번호, 전화번호, 주소, 표지어가 붙은 성명을 탐지하고 마스킹한다.
4. 탐지 시 사용자가 마스킹 원문을 확인해야 다음 단계로 진행할 수 있다.
5. 서버의 `/api/structure-items`가 같은 필터를 다시 실행한다. 클라이언트 검사를 우회한 개인정보가 감지되면 구조화 요청을 거부한다.

성명과 주소 탐지는 규칙 기반이므로 미탐과 과탐이 가능하다. 법률 준수를 보장하는 장치가 아니라 데이터 최소화와 사용자 확인을 위한 1차 방어선이다. 실제 운영 전 개인정보 처리방침, 처리위탁 여부, 공급자 데이터 보존 설정을 조직의 개인정보 담당자와 검토해야 한다.

## 네트워크 큐

STT 전송이 실패하면 녹음 Blob을 브라우저 IndexedDB에 임시 저장한다. 화면에 대기 건수를 표시하고 사용자가 재전송 또는 삭제할 수 있다. 서버에는 음성 파일을 저장하지 않는다.

## 사용량과 비용 추정

브라우저의 `digijosaVoiceMetrics` 로컬 스토리지에 최근 500개 이벤트를 보관한다. 녹음 길이, 호출 성공/실패, 토큰 수, 모델, 추정 비용, 등록 물건 수가 기록된다. 금액은 배포 환경의 가격 설정과 모델별 코드 테이블을 기반으로 한 참고치이며 청구서가 아니다.

콘솔에서 내려받기:

```js
copy(JSON.stringify(JSON.parse(localStorage.getItem('digijosaVoiceMetrics') || '[]'), null, 2))
```
